# SPEAR Framework

## 名称
Situation-Problem-Effect-Action-Result

## 结构
- **Situation**: 情境/背景
- **Problem**: 问题
- **Effect**: 影响
- **Action**: 行动
- **Result**: 结果

## 适用场景
- 说服性写作和演讲
- 营销文案创作
- 销售提案设计
- 政策倡导
- 投资者推介
- 产品发布演示

## 示例
**输入**: "写一份项目延期说明"

**优化后**:
```
Situation: 
我们的 SaaS 平台在 Q2 经历了用户量激增 300%，从 1 万用户增长到 4 万用户

Problem:
现有服务器架构无法支撑如此快速的增长，导致系统频繁宕机，平均每周 2 次服务中断

Effect:
- 客户满意度从 4.5 降至 3.2
- 3 个重要客户威胁要终止合同
- 品牌声誉受损，社交媒体上负面评价增加

Action:
1. 立即启动架构升级项目，迁移到微服务架构
2. 与 AWS 签订企业支持协议，确保 24/7 技术支持
3. 增加 3 名资深 DevOps 工程师
4. 实施自动化监控和告警系统

Result:
- 系统稳定性提升至 99.9%
- 客户满意度回升至 4.3
- 成功挽留所有重要客户
- 为未来增长打下坚实基础
```

## 使用提示
- 强调问题的紧迫性
- 行动要具体可衡量
- 结果要有数据支撑
