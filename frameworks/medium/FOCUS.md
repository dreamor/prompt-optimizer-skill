# FOCUS Framework

## 名称
Features-Objective-Constraints-User-Setup

## 结构
- **Features**: 功能/特性
- **Objective**: 目标
- **Constraints**: 约束条件
- **User**: 目标用户
- **Setup**: 设置/环境

## 适用场景
- 产品分析和比较
- 竞争对手研究
- 特性评估
- 采购决策支持
- 技术选型分析
- 投资评估

## 示例
**输入**: "评估我们的新功能"

**优化后**:
```
Features: 
- AI 驱动的代码自动补全
- 支持 20+ 编程语言
- 与 VS Code、IntelliJ 集成
- 离线模式支持

Objective: 
评估这个功能对开发团队生产力的提升效果，决定是否全面推广

Constraints:
- 预算限制：每开发者每月不超过 20 美元
- 安全要求：代码不能上传到外部服务器
- 时间限制：2 周内完成评估

User: 
- 主要用户：10 人后端开发团队
- 技术栈：Python、Go、JavaScript
- 经验水平：3-8 年

Setup:
- 现有工具：GitHub Copilot（试用期即将结束）
- 基础设施：公司内网开发环境
- 评估指标：编码速度、代码质量、开发者满意度

任务: 设计一个 2 周的试点评估计划，包括每日反馈收集和最终对比报告模板
```

## 使用提示
- 适合产品评估和对比场景
- 约束条件要尽早明确
